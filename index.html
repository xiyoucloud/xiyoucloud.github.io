sftp日志

2023-02-23
字数统计: 290   |   阅读时长≈ 1 分钟

stfp服务默认不记录日志到文件中，不过你可以通过修改配置文件来让它将日志记录在文件中。修改配置文件需要 root 权限。

修改配置文件/etc/ssh/sshd_config

注意：不要修改到配置文件的其他信息，否则可能导致无法登陆。

# 为防止改乱配置文件，需要先备份文件
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

# 编辑文件
sudo vi /etc/ssh/sshd_config

# 修改前
Subsystem       sftp    /usr/lib/openssh/sftp-server
# 修改后
Subsystem       sftp    /usr/lib/openssh/sftp-server -l VERBOSE -f LOCAL3

修改配置文件/etc/rsyslog.conf

# 在文件末尾增加一行
local3.*    /var/log/sftp.log

重启sshd和rsyslog

service sshd restart
service rsyslog restart

查看日志

tail -f /var/log/sftp.log

之后再通过sftp上传文件到服务器时，就可以到 /var/log/sftp.log 中查看日志了。以下图片是将本地的 cut.mp4 文件上传到服务器的 /home/txb 时文件中记录的日志信息，从日志中的 Permission denied 可以知道当前用户没有该目录的写入权限，因此上传文件失败了，最后会话关闭了（session closed）。

暂未归类的问题

2023-02-12
字数统计: 278   |   阅读时长≈ 1 分钟

libssl.so.1.0.0: cannot open shared object file: No such file or directory

这个动态链接库已经过时了，使用 conda 安装 openssl，然后给对应的动态链接库创建一个软链接即可。

base 环境的动态链接库通常保存在 ~/miniconda3/lib目录，其他虚拟环境的动态链接库通常保存在~/miniconda3/envs/xxx/lib目录中。

conda install openssl

# 以虚拟环境为 xxx 为例
cd ~/miniconda3/envs/xxx/lib
ln -s libssl.so libssl.so.1.0.0

# 然后重新执行软件

解决这个问题的原理是新版的 libssl.so 可能兼容旧版，所以给它创建一个别名使用。当然，如果不兼容，或许更好的方式是更新依赖旧的动态链接库的软件。

libEGL.so.1: cannot open shared object file: No such file or directory

apt-get install libegl-dev

configure: error: gsl-config not found, is GSL installed?

apt-get install libgsl-dev

GLPK is not available, Unimplemented function call

sudo apt-get install -y libglpk-dev

# 然后重装 igraph
install.packages("igraph", force = TRUE)

报错场景：

library("igraph")
g <- make_graph("Zachary")
oc <- cluster_optimal(g)

# 报错信息
Error in cluster_optimal(g) : 
  At optimal_modularity.c:85 : GLPK is not available, Unimplemented function call
